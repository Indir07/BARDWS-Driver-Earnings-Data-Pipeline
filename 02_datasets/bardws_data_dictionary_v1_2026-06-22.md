# BARDWS Dataset Documentation: Schemas, Data Dictionary, and Lineage
**Version:** v1  
**Release Date:** 2026-06-22  
**Author:** Antigravity Data Pipeline  

This document serves as the data dictionary, schema specification, and source lineage for the two normalized outputs generated by the BARDWS Data Pipeline.

---

## 1. Normalized Aggregator Earnings Table
**File Name:** `bardws_dataset_aggregator-earnings_v1.(parquet|csv)`  
**Purpose:** Normalizes and merges ride-level and weekly earnings summaries extracted from Uber, Ola, Rapido, and Namma Yatri PDF statements.

### 1.1 Schema & Data Dictionary

| Column Name | Data Type | Unit | Description | Notes / Rules |
| :--- | :--- | :--- | :--- | :--- |
| `aggregator` | string | — | The ride-hailing platform name | "Uber", "Ola", "Rapido", or "Namma Yatri" |
| `statement_period_start` | date | YYYY-MM-DD | Start date of the statement billing cycle | Parsed from PDF statement header |
| `statement_period_end` | date | YYYY-MM-DD | End date of the statement billing cycle | Parsed from PDF statement header |
| `txn_date` | date | YYYY-MM-DD | The transaction or activity date | `null` for Ola (only weekly data exists) |
| `trips` | integer | count | Number of rides completed on this date/week | Parsed from statement table |
| `rider_payment_inr` | float | ₹ | Gross amount paid by the customer/rider | Pre-commission, pre-fees |
| `platform_fee_inr` | float | ₹ | Total platform commission and booking fees | Net amount retained by the platform |
| `net_earnings_inr` | float | ₹ | Net payout received by the driver | Equals `rider_payment_inr - platform_fee_inr` (plus tips/incentives) |

### 1.2 Data Lineage & Transform Log
*   **Source System:** Monthly/weekly captain statements in PDF format under `assets/aggregator_pdfs/`.
*   **Ingest Timestamp:** 2026-06-22
*   **Transform Version:** v1
*   **Parser Details:**
    *   **Uber:** Extracts dates using `Statement period:` pattern. Captures daily rows. For v2019 layouts, `platform_fee_inr` is the service fee. For v2021/v2023, it is `service_fee + booking_fee`.
    *   **Ola:** Extracts period using `Month:` pattern. Captures weekly rows. `platform_fee_inr` matches `Commission (Rs.)`.
    *   **Rapido:** Extracts period using `Period:` pattern. Captures daily rows. `platform_fee_inr` is `Rapido Fee`.
    *   **Namma Yatri:** Extracts month/year. Captures daily rows. `platform_fee_inr` is the sum of `Platform Subscription` and `GST`.

## 2. Cleaned and Enriched Driver Dataset
**File Name:** `bardws_dataset_driver-enriched_v1.(parquet|csv)`  
**Purpose:** Cleans the anonymised driver daily activity log and enriches it with local weather data and retail fuel prices.

### 2.1 Schema & Data Dictionary

| Column Name | Data Type | Unit | Description | Notes / Rules |
| :--- | :--- | :--- | :--- | :--- |
| `driver_id` | string | — | Anonymised driver identifier | Values: `BARDWS-1000` to `BARDWS-1199` |
| `log_date` | date | YYYY-MM-DD | Logged activity date | Primary key part (Jan 2023 – Dec 2024) |
| `demand_zone` | string | — | Primary operating zone | e.g. Koramangala, MG Road, Indiranagar |
| `hours_worked` | float | hours | Total hours worked on that day | `0.0` on sick days and off days |
| `km_driven` | float | km | Total kilometers driven | Includes deadheading. Imputed if missing |
| `rides_completed` | integer | count | Trips completed by driver on this day | Zero-activity days are `0` |
| `gross_fare_inr` | float | ₹ | Total fares collected from passengers | Pre-fuel, pre-commission. Imputed if missing |
| `fuel_topped_up_litres` | float | litres | Quantity of fuel refueled on this day | Lumpy (often `0.0` on non-refueling days) |
| `sick_day` | integer | 0/1 | Flag indicating self-reported sick day | `1` = Sick (activity fields are zeroed) |
| `hours_outlier` | integer | 0/1 | Outlier flag for impossible work shifts | `1` = `hours_worked` equals `26` (invalid log) |
| `rainfall_mm` | float | mm | Total daily rainfall recorded by IMD station | Joined from weather dataset |
| `temp_max_c` | float | °C | Maximum temperature recorded on this day | Joined from weather dataset |
| `temp_min_c` | float | °C | Minimum temperature recorded on this day | Joined from weather dataset |
| `humidity_pct` | float | % | Relative humidity percentage | Joined from weather dataset (contains nulls) |
| `retail_price_inr_per_litre` | float | ₹/L | Retail price of IndianOil Petrol in Bengaluru | Joined from fuel prices dataset |

### 2.2 Data Lineage & Quality Rules
*   **Source Systems:**
    1.  Driver Logs: `assets/driver_logs/member_driver_logs_sample.csv`
    2.  Bengaluru Weather: `assets/weather/imd_bengaluru_weather_2018_2024.csv` (IMD Station: `Bengaluru City (IMD 43295)`)
    3.  Bengaluru Fuel Prices: `assets/fuel_prices/bengaluru_fuel_prices_2018_2024.csv`
*   **Ingest Timestamp:** 2026-06-22
*   **Transform Version:** v1
*   **Data Cleaning & Imputation Rules:**
    1.  **Deduplication:** 80 exact duplicate rows sharing `driver_id` and `log_date` were identified and removed (`keep='first'`).
    2.  **Outlier Flagging:** 12 impossible rows showing `hours_worked = 26` are flagged with `hours_outlier = 1` for downstream exclusion.
    3.  **Missing `km_driven` Imputation (1.5% missing):**
        *   If `sick_day == 1` or `hours_worked == 0`, `km_driven` is imputed as `0.0`.
        *   If `hours_worked > 0`, `km_driven` is imputed as `hours_worked * driver_median_speed`, where `driver_median_speed` is the driver's median km per hour on active days. If a driver has no active logs, the global median speed (17.51 km/h) is used.
    4.  **Missing `gross_fare_inr` Imputation (0.6% missing):**
        *   If `sick_day == 1` or `hours_worked == 0`, `gross_fare_inr` is imputed as `0.0`.
        *   If `hours_worked > 0`, `gross_fare_inr` is imputed as `rides_completed * driver_median_fare_per_ride`, where `driver_median_fare_per_ride` is the driver's median fare per completed trip. If no driver data exists, the global median fare per ride (₹122.56) is used.
    5.  **Enrichment:** Left-joined weather and fuel prices by `log_date == date`. Fuel prices are filtered to `oil_company == 'IndianOil'` and `fuel_type == 'Petrol'` to provide a consistent cost index.
